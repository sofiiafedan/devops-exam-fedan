# Task 3: Application Deployment (Kubernetes + Nginx)

У цьому завданні реалізовано автоматизоване розгортання застосунку в кластері Kubernetes за допомогою GitHub Actions. 

**Архітектурне рішення:**
1. **Source Code:** Вихідний код застосунку та `Dockerfile` не дублюються в цьому репозиторії. CI/CD пайплайн автоматично клонує актуальний код з цільового репозиторію під час збірки.
2. **Environments:** Створено два незалежні середовища (Dev та Release) з використанням єдиного кластера Minikube.
3. **Networking (Вимоги методички):** Застосунок доступний на портах `8002` (Dev) та `8003` (Release). Оскільки Kubernetes `NodePort` не підтримує порти нижче 30000, було налаштовано `Nginx` як Reverse Proxy на самому сервері. Nginx слухає порти 8002/8003 і маршрутизує трафік на внутрішні порти кластера (30002/30003).

Усі маніфести та конфігурації генеруються та застосовуються автоматично через пайплайн `.github/workflows/app-deploy.yml`. Файл `dual-env.yaml` у цій директорії наведено для демонстрації структури ресурсів Kubernetes.
